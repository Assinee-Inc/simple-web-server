# Feature Specification: Omit Client's Date of Birth from CSV Export

**Feature Branch**: `[###-feature-name]`  
**Created**: 2026-04-23
**Status**: Draft  
**Input**: User description: "A data de nascimento do cliente não deve ser exportada para o CSV."

## User Scenarios & Testing *(mandatory)*

### User Story 1 - Export client data without sensitive information (Priority: P1)

As a user with permission to export client data, I want to be able to export a list of clients to a CSV file, so that I can use this data for external analysis, without exposing sensitive information like the date of birth.

**Why this priority**: This is the core functionality of the feature and is essential for protecting user privacy.

**Independent Test**: A user can trigger the client data export and verify that the generated CSV file contains all the expected client information, except for the date of birth.

**Acceptance Scenarios**:

1. **Given** a user is logged in and has permission to export client data, **When** the user navigates to the client list and clicks the "Export to CSV" button, **Then** a CSV file is downloaded.
2. **Given** the CSV file from the previous scenario, **When** the user opens the file, **Then** the file contains columns for client name, email, and phone number, but not for the date of birth.

### Edge Cases

- What happens when the client list is empty? The exported CSV should contain only the header row.
- What happens if a client has missing information (e.g., no phone number)? The corresponding cell in the CSV should be empty.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: The system MUST provide a mechanism to export client data to a CSV file.
- **FR-002**: The exported CSV file MUST NOT include the `birthdate` column or data.
- **FR-003**: The exported CSV file MUST include the client's `name`, `email`, and `phone` number.
- **FR-004**: The CSV file must be well-formed and readable by standard spreadsheet software.

### Key Entities *(include if feature involves data)*

- **Client**: Represents a client in the system. Key attributes for this feature are `name`, `email`, `phone`, and `birthdate`.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 100% of client data CSV exports generated by the system do not contain the "birthdate" column.
- **SC-002**: The time to generate the CSV export for 10,000 clients should be less than 30 seconds.

## Assumptions

- The functionality to export client data to CSV already exists. This feature is about modifying the existing functionality.
- Users who can export data are already defined and have the necessary permissions.
- The fields `name`, `email`, and `phone` are available for all clients.
