# Implementation Plan: Remove Client ↔ Creator Relationship

**Branch**: `feature/002-remove-client-creator-relationship` | **Date**: 2026-04-24 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/002-remove-client-creator-relationship/spec.md`

## Summary

Remove the `client_creators` join table and `Creators` field from `Client` model. The relationship between Client and Creator will be derived exclusively from purchases (Client → Purchase → Ebook → Creator). This eliminates data redundancy and prevents inconsistent states where a client is associated with a creator but has no purchase.

## Technical Context

**Language/Version**: Go 1.21+
**Primary Dependencies**: GORM, Chi Router
**Storage**: SQLite (dev) / PostgreSQL (prod)
**Testing**: `go test` (unit + integration tests)
**Target Platform**: Linux server (web service)
**Project Type**: Web service / REST API
**Performance Goals**: No degradation — purchase-based queries must be as efficient as join-table queries
**Constraints**: Existing client data must be preserved; no production data loss
**Scale/Scope**: Affects all client listing/export features; 1 model deleted, ~8 methods updated

## Phase 1: Design & Contracts

### Data Model Changes

**Client** (`internal/sales/model/client.go`)
- REMOVED: `Creators []*accountmodel.Creator` (gorm many2many)
- RETAINED: `ID`, `PublicID`, `Name`, `CPF`, `Birthdate`, `Email`, `Phone`, `Validated`, `Purchases`

**ClientCreator** (`internal/sales/model/client_creator.go`)
- DELETED: Entire model and `client_creators` table

### Query Changes

| Method | Old Approach | New Approach |
|--------|--------------|--------------|
| `FindClientsByCreator` | `JOIN client_creators` | `JOIN purchases + ebooks` |
| `FindByClientsWhereEbookNotSend` | `JOIN client_creators` | `JOIN purchases` (NOT IN subquery) |
| `FindByClientsWhereEbookWasSend` | `JOIN client_creators` | `JOIN purchases` (IN subquery) |
| `Save` | Creates `client_creators` entries | No association logic |
| `FindByIDAndCreators` | Subquery with `client_creators` | Simplify to purchase-based |

This is an internal data model refactor — no external API changes, no file format changes, no external integrations affected.
